# Enterprise Lakehouse Storage Zoning using Azure Data Lake Storage Concepts

## Project Overview
This project simulates how Azure Data Lake Storage can be used as the storage foundation for a Lakehouse architecture.

It is based on an enterprise pattern where data is organized into layered storage zones so that raw source data, cleaned and standardized data, and curated business-ready outputs are stored in a scalable and traceable way.

The project includes:
- raw zone creation
- standardized zone creation
- curated zone creation
- lineage logging
- storage summary output
- storage zone design metadata

## Business Goal
The goal of this project is to show how layered storage design improves:
- source traceability
- auditability
- data reuse
- data quality control
- centralized storage for analytics, ML, and AI-ready use cases
